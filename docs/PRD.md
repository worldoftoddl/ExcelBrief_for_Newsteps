## 1. 배경과 문제

회계 감사 업무의 산출물 대부분은 Excel 형식의 감사조서(workpaper)로 작성된다.
신입(뉴스텝) 회계사는 조서의 구조·목적·수행된 절차를 파악하는 데 오랜 시간이 걸리고,
선임의 구두 설명에 의존한다. 이 지식 전달을 AI 에이전트로 보조한다.

**본 프로젝트는 회계법인 지원용 포트폴리오 MVP다.** 실무 배포가 아니라
HuggingFace Spaces 공개 데모로 역량을 입증하는 것이 1차 목적이며,
보안·인증·권한은 고려 대상이 아니다. 데모용 조서는 가상 데이터로 제작한다
(공개 URL에 실제 피감사회사 데이터를 올릴 수 없으므로).

## 2. 목표

조서 Excel 파일을 에이전트에게 전달하면, 에이전트가 파일을 읽고
**한국 회계감사기준·K-IFRS·회계감사실무지침에 근거하여** 텍스트로 설명한다.

핵심 설명 범위:

1. **가능한 감사절차** — 이 조서(계정/영역)에서 어떤 감사절차가 가능한지
2. **수행된 절차 해석** — 이미 작성된 조서라면 어떤 감사절차가 이루어졌는지
3. **추가 필요 절차** — 미완성 조서라면 어떤 절차가 추가로 필요한지

모든 기준 인용은 auditPaper_MCP의 복합 ID(cid, 예 `KIFRS::1115::31`)로
원문 대조가 가능해야 한다.

## 3. 사용자

- **0차**: 회계법인 채용 담당자·면접관 — 공개 데모 URL로 열람 (포트폴리오 본연의 목적)
- **1차**: 뉴스텝(신입) 회계사 — 조서 학습·이해 목적 (제품 시나리오)
- **2차**: 인차지/선임 — 조서 리뷰 보조, 절차 누락 점검

## 4. 요구사항

### 기능 (MVP)

| ID | 요구사항 |
|---|---|
| F1 | 챗 형태 웹 UI에서 대화로 조서 설명을 요청·수신 |
| F2 | 감사조서뿐 아니라 **범용 Excel 파일**도 문제없이 읽고 설명 |
| F3 | 설명에 회계감사기준·K-IFRS 근거를 cid와 함께 인용 |
| F4 | 대규모 워크북도 처리 — 에이전트가 도구로 필요한 부분만 탐색하며 읽기 |
| F5 | 모델 라우팅 — 상용 API(Anthropic)와 로컬 모델을 설정으로 전환 |

### 비기능

| ID | 요구사항 |
|---|---|
| N1 | **공개 데모 전제** — HuggingFace Spaces 배포. 포트폴리오 MVP이므로 보안·인증은 비고려. 데모 조서는 가상 데이터 사용 |
| N2 | 언어는 Python. 단, 에이전트 도구 내부에서 타 언어/외부 프로세스 호출 허용 |
| N3 | 기존 자산 재사용 — UI는 braincrew-lab/agent-chat-ui 포크, RAG는 auditPaper_MCP 완성본(HF Space 상시 배포) |

## 5. MVP 범위 제외 (백로그)

- 챗 UI에서 xlsx 직접 업로드 (UI가 기본 지원하는 형식은 이미지·PDF뿐 → MVP는
  가상 샘플 조서를 미리 배치해 선택하는 방식. **공개 데모 가치가 커서 백로그 최우선**)
- SpreadsheetLLM류 시트 압축 인코딩 (초대형 워크북 대응)
- LiteLLM 게이트웨이 (모델 자동 폴백·사용량 통제)
- 다중 사용자 인증·권한

## 6. 성공 기준

1. 샘플 조서를 지정해 "해석해줘" 요청 시, 수행 절차 + 근거 기준(cid 인용) +
   추가 필요 절차가 담긴 설명이 생성된다
2. 생성된 해석을 auditPaper_MCP의 `eval/score_interpretation.py` 채점 체계로 평가 가능하다
3. 범용 Excel(비감사 파일)도 구조 요약·내용 설명이 동작한다
4. UI에서 모델(상용/로컬)을 전환해도 동일 대화 흐름이 유지된다
5. **HuggingFace Spaces 공개 URL에서 위 1~4가 데모 가능하다** — 링크 하나로
   포트폴리오 열람이 완결된다
