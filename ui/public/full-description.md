# ExcelBrief for Newsteps

**신입(뉴스텝) 회계사를 위한 감사조서 해설 에이전트**입니다.
Excel 감사조서를 직접 읽고, 한국 회계감사기준·K-IFRS·회계감사실무지침에 근거해 설명합니다.

## 🎯 무엇을 해주나요

1. **가능한 감사절차 설명** — 조서 서식을 보고 이 조서로 어떤 감사절차를 수행할 수 있는지 알려줍니다.
2. **수행된 절차 해석** — 작성된 조서의 값·수식·메모·서식(색상 표시 등)을 근거로 무엇이 수행되었는지 해석합니다.
3. **추가 필요 절차 제안** — 미완성 조서에서 아직 비어 있는 항목과 추가로 필요한 절차를 짚어줍니다.

감사조서가 아닌 범용 Excel 파일도 구조를 탐색해 설명할 수 있고,
Word(.docx) 문서(내부통제 기술서·회의록 등)도 읽을 수 있습니다.

## 📎 내 파일 업로드

클립 버튼을 누르거나 파일을 채팅창에 끌어다 놓으면 **Excel(xlsx/xlsm/xls)·Word(docx)**
파일을 첨부할 수 있습니다 (최대 20MB). 첨부 후 "이 파일 설명해줘"처럼 질문하세요.

> 업로드 파일은 데모 서버의 임시 저장소에 보관되며 서버 재시작 시 삭제됩니다.
> **실제 피감사회사 데이터는 절대 업로드하지 마세요.**

## 📂 미리 준비된 조서

한국공인회계사회 공식 감사조서 서식 5종과 가상 데모 조서 3종이 준비되어 있습니다.

| 파일 | 내용 |
|---|---|
| 1100~1300 | 감사계약 |
| 3100~3800 | 위험에 대한 대응 |
| 3650 | 감사 전 재무제표 확인 |
| 3900·3900A | 핵심감사사항 |
| 4000 | 계정별 실증절차 (K-IFRS용) |
| 데모조서 5300 | 현금및현금성자산 — 작성 완료 예시 (가상) |
| 데모조서 5400 | 매출채권 — 작성중 예시 (가상) |
| 데모 예산집행 현황 | 감사조서가 아닌 범용 Excel 예시 (가상) |

첫 화면의 예시 질문 버튼을 누르거나, 조서 번호를 언급하며 자유롭게 질문하세요.

## 📖 답변의 근거

- **셀 주소 인용**: 조서에 대한 주장은 `3650!B4`처럼 실제 셀 주소를 근거로 제시합니다.
- **기준서 인용**: 회계·감사 기준 관련 설명에는 `KIFRS::1115::31`류의 문단 식별자를 병기합니다.
  기준서 검색은 별도 RAG 서버(auditPaper MCP)를 통해 원문을 조회합니다.

## 💡 사용 팁

- **연속 질문**: 같은 대화에서는 맥락이 유지됩니다. "방금 그 조서에서 ~는?"처럼 이어서 물어보세요.
- **도구 호출 확인**: 렌치 아이콘을 켜면 에이전트가 조서의 어느 시트·범위를 읽었는지 과정을 볼 수 있습니다.
- **구체적으로**: 조서 번호나 계정명(예: 매출채권)을 함께 언급하면 더 정확한 답을 받습니다.

## 🛠️ 기술 스택

LangChain/LangGraph 에이전트 + 멀티 벤더 LLM(Claude 기본 · GPT · Gemini · 로컬 Ollama,
입력창 옆 드롭다운에서 선택) · Excel 탐색 도구 7종(openpyxl·xlrd) ·
Word 문서 도구(python-docx) · 기준서 RAG(auditPaper MCP, HuggingFace Space) · agent-chat-ui
